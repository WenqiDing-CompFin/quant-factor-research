# 量化研究实习日报模板

> 提交前删除方括号中的提示。日报的目标是让 Mentor 在一分钟内看懂：今天验证了什么、具体改了什么、结果如何、哪里仍有风险、明天如何验收。不要只写“阅读论文”“构造因子”或“运行回测”。

## 今日日报：YYYY-MM-DD

### 1. 今日研究思路

[用 1-3 句话写清研究问题、可证伪假设、数据、方法和判断标准。]

```text
基于 [昨日结果/论文或研报中的具体方法/代码问题]，今天验证 [具体假设]。
使用 [synthetic 或真实公开数据、日期范围、股票数、频率]，按照 [因子公式、窗口、滞后、分组或对照方法] 处理，
并以 [Rank IC、ICIR、分组收益、benchmark excess return、换手、成本敏感性或测试结果] 作为判断标准。
```

### 2. 关键进展

[按真实完成顺序写，每条包含对象、方法、参数和产出位置。]

1. 数据：[检查了哪些字段、样本范围、重复值、缺失值、逐 ticker 月度连续性、价格有效性或发布日期规则。]
2. 因子：[实现或修改了哪个因子；公式、方向、窗口和滞后是什么。]
3. 检验：[运行了 Rank IC、benchmark、成本敏感性或哪一项对照实验。]
4. 工程：[修改的模块/函数；新增或复跑的测试；输出文件。]
5. 提交：[commit hash、PR/MR、研究笔记或结果图位置。]

### 3. 结果

[必须写结果。负面、不显著或尚不能判断也是结果。所有模拟结果明确标注 synthetic。]

```text
- 数据结果：[有效日期数、股票数、有效截面数、缺失率、剔除数量]
- 因子结果：[mean Rank IC、ICIR、valid-observation positive IC ratio、Q1-Q5 收益、top-minus-bottom spread、与基线差异]
- 组合结果：[net return、benchmark、excess return、turnover、drawdown；注明 synthetic/真实公开]
- 成本结果：[0/5/10/20/50 bps 下结果如何变化]
- 工程结果：[pytest 通过/失败数量、运行时间、生成的 CSV/PNG]
- 当前判断：[支持/不支持/证据不足以判断原假设；原因和适用边界]
```

### 4. 遇到的问题与下一步

[问题必须包含影响、排查动作、当前判断和明日验收标准。]

```text
- 问题：
- 对研究结论或代码的影响：
- 今日已完成的排查：
- 当前判断：
- 尚未排除的解释：
- 明日动作：
- 明日验收标准：
```

## 提交前检查

- 是否提出了一个具体、可验证的问题？
- 是否说明数据性质、样本范围、公式、参数和滞后规则？
- 是否给出数字、对照或测试结论，而不是只写“已完成”？
- 是否区分 synthetic 输出与真实市场证据？
- 是否记录失败、返工、异常或证据不足？
- 是否写清下一步的可验收产出？
- 是否附上代码、结果和 commit 位置？
- 是否通过保密检查？
